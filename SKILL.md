# 高端技术候选人中文简历生成器

## When to Use This Skill

Use this skill when the user needs to generate a concise Chinese resume-style candidate profile for business leaders, hiring managers, interviewers, or internal recruitment discussions.

Typical use cases include:

- 候选人没有正式简历，或不愿意准备简历
- 用户基于一手访谈记录，为业务快速整理候选人经历
- 用户提供 LinkedIn 截图、个人主页、Google Scholar、GitHub、论文列表等碎片化材料
- 用户希望把候选人材料整理成可直接交付业务或面试官的中文简历
- 用户需要为高端 AI/ML 人才生成推荐材料、候选人简历或内部评审材料

This skill is especially useful for:

- AI Research Scientist
- Applied Scientist / ML Engineer
- AI Infrastructure Engineer
- Pretraining Infra Engineer
- Post-training Engineer
- RL Infra Engineer
- Multimodal Researcher
- Agent Researcher / Engineer
- AI Product / Technical Leader

## Activation Interaction

When this skill is invoked, first check whether the user has already provided enough candidate information.

If the user has already pasted raw notes, screenshots, profile text, or candidate highlights, do not ask them to refill a template. Directly extract the information and generate the resume deliverable.

If the user has not provided enough information, ask the following in Chinese:

> 请问你是否已经准备好以下材料？不需要全部都有，能提供多少就提供多少。我会基于已有材料直接生成一版中文简历成品。
>
> 1. 访谈记录 / 沟通记录
> 2. LinkedIn 截图或主页内容
> 3. 候选人个人主页 / lab page / company bio
> 4. Google Scholar / 论文列表
> 5. GitHub / 开源项目
> 6. 你希望推荐到的方向或岗位
> 7. 你希望重点强调的 1–3 条候选人亮点

Then provide this fill-in template only when useful:

```md
# 候选人材料填写模板

## 1. 基本信息
- 姓名：
- 当前公司：
- 当前职位：
- 所在地：
- 当前职级 / seniority（如已知）：
- 意向度 / 开放程度：
- 推荐方向 / 目标岗位：

## 2. 访谈记录 / 沟通记录
请直接粘贴原始记录，不需要整理得很完美。

可包括：
- 当前团队 / 工作方向：
- 代表项目：
- 技术细节：
- 团队规模 / 协作对象：
- 候选人表达出的兴趣点：
- 地点 / 回国意向等信息：

## 3. 公开履历信息
可粘贴 LinkedIn 截图 OCR 内容、个人主页文字、Google Scholar、GitHub 或论文列表。

可包括：
- 工作经历：
- 教育经历：
- 技能标签：
- 研究方向：
- 代表论文 / 开源项目：

## 4. 推荐重点
- 最希望突出的亮点 1：
- 最希望突出的亮点 2：
- 最希望突出的亮点 3：
- 哪些内容不要写：
- 输出格式偏好：默认中文简历版 / 简版推荐理由 / 完整评估版：
```

Rules:
- If the user provides partial information, proceed with available materials.
- If a field is missing and not necessary, omit it rather than writing `待确认`.
- If the user asks for a quick version, produce a concise draft first.
- If the user provides raw notes without using the template, do not force the template.

## What This Skill Does

1. **生成中文简历成品**
   - 默认直接输出可交付给业务、面试官或内部评审使用的中文候选人简历。
   - 不输出写作过程说明、材料来源说明、面向对象说明或语言说明。

2. **提炼用户给定亮点**
   - 优先使用用户提供的推荐理由、候选人亮点和业务判断。
   - 尤其是“推荐语 / 一句话推荐”部分，必须优先来自用户给定的人选亮点。
   - 不自行扩写成过度包装或咨询报告式表达。

3. **重构候选人经历**
   - 基于访谈记录、LinkedIn、个人主页、Google Scholar、GitHub 等材料，整理教育背景、工作经历、核心职责和代表贡献。
   - 输出应像用户手写的业务验证版中文简历，而不是标准英文 resume 或长篇 candidate briefing。

4. **保持简洁可读**
   - 工作经历默认写得简洁，核心经历 3–5 个 bullet，次要经历 1–2 个 bullet。
   - 不拆成 Scope、Key Contributions、Technical Keywords、Core Value 等复杂报告结构。

5. **保持事实克制**
   - 不编造职级、时间、ownership、论文贡献、薪资、汇报线或影响指标。
   - 不确定信息不要强行写入简历正文。

## Core Principles

### 1. 默认交付中文简历成品

默认交付的是“中文简历成品”，不是分析报告、不是候选人画像报告、不是面试策略包。

除非用户明确要求，不要输出以下内容：

- 材料来源说明，例如“由 Jeff 沟通一手访谈 + LinkedIn 公开履历整合”
- 面向对象说明，例如“面向：预训练 Infra 负责人 / RL Infra 负责人”
- 语言说明，例如“语言：Simplified Chinese / 关键技术词保留英文”
- 候选人画像
- 技术能力拆解
- 目标岗位匹配度评估
- 面向不同岗位负责人的分方向评估
- 待确认信息清单
- 关键情报
- 招聘官总结 / Final Pitch
- 建议面试问题

默认输出应参考用户认可的样本风格：李尚文中文简历、Le Chen简历、沈卓然简历、黄杰简历、林旭东简历。

---

### 2. Output Language

Unless the user explicitly requests another language, all final outputs generated by this skill must be written in Simplified Chinese.

Use Chinese for:

- 推荐语
- 个人基本信息
- 教育背景
- 工作经历
- 核心职责
- 论文 / 专利 / 开源项目说明

Keep English technical terms when they are standard AI/ML industry terms or when translation may reduce precision.

Examples:

- pretraining
- post-training
- SFT
- RLHF / RLVR
- inference
- agent infra
- RL infra
- RL environment scaling
- GPU kernel
- distributed training
- model serving
- benchmark
- evaluation
- multimodal understanding
- world model
- coding agent
- computer use agent
- CUDA / PTX
- JAX / XLA
- PyTorch / TorchInductor
- NCCL
- GB200 / GB300 / H100

Do not output English section titles unless the user specifically asks for an English version.

---

### 3. Evidence First

Do not fabricate facts, titles, dates, education, project ownership, publications, impact, or technical contributions.

Only use information that is:

- Explicitly provided by the user
- Visible in the provided screenshots or materials
- Clearly supported by public sources
- Reasonably inferable from the available evidence

When ownership is unclear:

- Use “参与” rather than “主导”
- Use “支持” rather than “负责”
- Use “主要参与 / 参与建设 / 参与优化” when contribution depth is not fully confirmed
- Use “具体 ownership 待进一步确认” only if the user asks for a full evaluation version

Never present uncertain information as confirmed fact.

---

### 4. 推荐语优先使用用户给定亮点

If the user provides 1–3 candidate highlights, use them as the primary material for the recommendation section.

Rules:

- 不要脱离用户给定亮点自行重写成很泛的推荐语。
- 可以优化表达，但不要改变用户原意。
- 可以保留用户的技术判断和业务判断。
- 推荐语应突出“为什么值得聊 / 推荐到什么方向 / 核心稀缺性”。
- 推荐语可以是一段，也可以是 2 条编号理由。

Preferred format:

```md
## 推荐语

（1）[亮点一]：……
（2）[亮点二]：……
```

If the user explicitly asks for “一句话推荐”, use:

```md
## 一句话推荐

[一句简洁推荐语]
```

### 6. 工作经历简洁优先

Professional experience should be concise and close to the user's manually written reference resumes.

Use reverse chronological order.

For each experience, use this format:

```md
[公司] | [职位]
[时间]
核心职责：
● [职责 / 贡献 1]
● [职责 / 贡献 2]
● [职责 / 贡献 3]
```

Writing rules:

- 每段经历默认 2–5 个 bullet。
- 优先写与推荐方向相关的经历。
- 早期或相关性较弱的经历可以压缩为 1–2 个 bullet。
- 不要拆成 Scope、Key Contributions、Technical Keywords、Core Value 四段。
- 不要单独写“对目标岗位的核心价值”。
- 不要写过多组织八卦、汇报线、内部稳定度、竞争情报。
- 如果用户给了明确技术表述，尽量沿用用户的技术判断和关键词。
- “负责 / 主导 / 参与 / 支持”必须和用户证据保持一致。

### 7. Reference System

Before generating the final resume, use the reference system to align the output with the user's approved resume style.

The reference system has four parts:

1. **Style Guide**
   - Defines the overall writing style, structure, tone, and default exclusions.

2. **Approved Resume Examples**
   - Represents user-approved resume samples that have been validated by business stakeholders.
   - Default style references include: 李尚文中文简历、Le Chen简历、沈卓然简历、黄杰简历、林旭东简历。

3. **Pattern Library**
   - Provides reusable phrasing patterns for recommendation sections, work experience bullets, and technical contribution descriptions.

4. **Negative Examples**
   - Defines formats and sections that should not appear in the default resume output.

When generating a resume, follow this priority order:

1. User-provided candidate highlights and raw notes
2. User-approved reference resume style
3. This skill's default output structure
4. General resume writing conventions

If there is a conflict between generic resume conventions and the user's approved reference style, follow the user's approved reference style.

### 9. Approved Resume Example Patterns

When imitating approved examples, focus on structure and density, not exact wording.

Approved examples typically follow these patterns:

#### 推荐语 Pattern

```md
## 推荐语

（1）[核心亮点一]：[基于用户给定材料，说明该候选人在目标方向上的稀缺性和相关经历]
（2）[核心亮点二]：[基于用户给定材料，说明该候选人的代表项目、头部平台经验或技术复合度]
```

Rules:
- 推荐语默认 1–2 条。
- 不要写成长篇销售话术。
- 不要加入用户没有提供的薪资、汇报线、组织情报或竞争窗口。

#### 工作经历 Pattern

```md
[公司] | [职位]
[时间]
核心职责：
● [用一句话说明主要负责方向]
● [用一句话说明代表项目 / 技术贡献]
● [用一句话说明模型、系统、产品或业务落地]
```

Rules:
- 核心经历 3–5 条 bullet。
- 次要经历 1–2 条 bullet。
- 早期经历可以合并或压缩。
- 不要拆成 Scope、Key Contributions、Technical Keywords、Core Value。

#### 技术表达 Pattern

Use concise technical phrases such as:

- 参与 [模型/系统] 的 pretraining infra 建设，重点支持 [distributed training / fault tolerance / parallelism / checkpoint / performance optimization] 等方向。
- 参与 [模型] 的 post-training 工作，覆盖 SFT / RLHF / RLVR / reward modeling / evaluation / data construction 等环节。
- 参与 agent infra / RL environment scaling 建设，支持 coding agent / computer use agent / tool use 等场景。
- 参与 image generation / video generation / multimodal understanding 相关模型训练与产品化，重点解决一致性、可控性、编辑能力或长视频连贯性等问题。

Only use these patterns when supported by user-provided evidence.

### 10. Negative Reference Examples

The following patterns should not appear in the default resume output unless the user explicitly asks for a complete evaluation version.

#### Do not output source or audience notes

Avoid:

```md
> 由 Jeff 沟通一手访谈 + LinkedIn 公开履历整合
> 面向：预训练 Infra 负责人 / RL Infra 负责人
> 语言：Simplified Chinese / 关键技术词保留英文
```

Reason: This is briefing metadata, not resume content.

#### Do not output candidate portrait by default

Avoid:

```md
## 候选人画像（3 分钟速览）
```

Reason: This makes the document look like a candidate assessment report rather than a resume deliverable.

#### Do not output technical capability breakdown by default

Avoid:

```md
## 技术能力拆解

**预训练 / 训练 Infra**
**推理 / Inference Infra**
**Agent Infra / RL Environment**
```

Reason: This is useful for analysis, but too heavy for the default resume version.

#### Do not output fit assessment by default

Avoid:

```md
## 目标岗位匹配度评估

### Strong Match
### Partial Match
### Risk / Gap
```

Reason: Fit assessment belongs to the complete evaluation version, not the default resume.

#### Do not output Final Pitch by default

Avoid:

```md
## 招聘官总结（Final Pitch）
```

Reason: This reads like sales language and should not appear in the default resume output.

---

### 11. Reference System Maintenance

When the user provides new approved samples or feedback, update the reference system rather than only changing one output.

Types of updates:

- If the user says a generated section is too long, update the style guide and length control rules.
- If the user says a format is business-validated, add it to the approved resume patterns.
- If the user says a section should not appear, add it to negative reference examples.
- If the user repeatedly uses a phrasing pattern, add it to the pattern library.

The goal is to make future outputs increasingly close to the user's own writing style.

## Writing Style

Use:

- 简体中文作为默认输出语言
- 中文简历体，而不是英文 resume 体
- 清晰、直接、业务可读的表达
- 用户提供的人选亮点作为推荐语主材料
- 必要的 AI/ML 英文技术术语
- 短 bullet，少长段落
- 接近用户手写样本的简洁风格

Avoid:

- 输出材料来源说明、面向对象说明、语言说明
- 咨询报告式结构
- 候选人画像、技术能力拆解、匹配度评估、关键情报、Final Pitch 等默认模块
- 过度包装和夸张形容词
- 未验证的薪资、汇报线、组织八卦或竞争信息
- 把不确定内容写成事实
- 把工作经历写成复杂的分层分析报告

Preferred tone:

- 专业
- 克制
- 简洁
- 业务可读
- 适合直接复制给业务或面试官

## Quality Checklist

Before finalizing the output, verify:

- 是否直接交付了中文简历成品，而不是分析报告？
- 是否删除了材料来源说明、面向对象说明和语言说明？
- 推荐语是否优先使用了用户提供的人选亮点？
- 个人基本信息中是否默认删除了当前年薪、汇报线、目标推荐方向、信息来源？
- 是否没有输出候选人画像、技术能力拆解、岗位匹配度、待确认信息、关键情报、Final Pitch、建议面试问题等默认不需要的模块？
- 工作经历是否足够简洁，接近用户手写样本风格？
- 是否保留了关键技术词，但没有过度展开？
- 是否避免把不确定内容写成事实？
- 是否可以直接复制给业务或面试官使用？
- 是否参考了用户认可的样本风格，而不是泛化成标准英文 resume 风格？
- 是否避免了 negative reference examples 中列出的默认禁用模块？
- 是否优先使用用户给定亮点，而不是自行创造新的推荐逻辑？

## Privacy and Sensitivity

Candidate information may be sensitive.

Do not include:

- Private contact information unless explicitly provided for this purpose
- Compensation details unless the user explicitly asks to include them
- Reporting line unless the user explicitly asks to include it
- Personal identity details unrelated to hiring
- Speculative personal judgments
- Sensitive attributes such as health, religion, political views, family status, or ethnicity
- Organization gossip or competitive intelligence unless the user explicitly requests an evaluation or strategy version

Use only job-relevant information.
